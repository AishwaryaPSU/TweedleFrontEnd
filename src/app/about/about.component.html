agm37@psu.edu

Assumptions


  • Twitter streaming API service supplies the tweets in Real Time
  • Ideal network conditions
  • Servers have required system configuration
  • Push notification service delivers notification in Real Time

Tech Stack

  • Open Source Libraries - Apache Kafka, Apache Storm,HBC client
  • Data Source - Twitter Streaming API
  • Backend Service - Play Framework
  • Frontend - Angular 2
  • Storage - MongoDB
  • Third Party Host Providers - Linode Servers
  • third party push notification service - PushWoosh


Overall Picture

Advantages

  • Different data sources can be integrated
  • Multiple producers and consumers can be added to Kafka with ease
  • All components can work in distributed mode, hence scalable.

Drawbacks

  • Storm needs relatively powerful servers at high data flow.
  • Needs complex exception handling logic

Improvements

  • Machine learning model for sentiment analysis
  • Android/iPhone push notification mobile application
  • Better Data Management
  • Better Websocket Handling
  • Better Threadpool Management
  • Better Deployment Mechanism

Features

  • Retry Mechanism on message delivery failure at Producer or Consumer
  • Asynchronous and non-blocking execution where needed
  • WebSocket Connection to stream live data to user interface
